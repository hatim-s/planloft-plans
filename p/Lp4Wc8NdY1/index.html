AI App Builder — N22 Editing Ergonomics Plan

DSL-expressible ergonomics improvements for styles, bindings, and events.

Decisions locked 2026-07-24. The agent co-authoring invariant remains intact.

Bug-tier fixes first

  • Replace the padding/gap raw-JSON textarea with a typed stepper and token detents.
  • Give event parameters visible labels through labeled FieldShell fields.

Decisions

  • Styles: Figma-style compact token clusters: segmented icon controls for small enums, tone/surface swatches, padding/gap stepper, radius glyphs, and human labels. Regroup Layout, Appearance, Typography, and Advanced through additive catalog metadata; use sanitizer-driven type-aware widgets for styleObject.
  • Preview: hover preview is a local-only override context, never persisted, avoiding LWW and agent races; selection commits it.
  • Bindings: field-frame token pill with icon, friendly label, direction glyph, and popover editor. Use “Read”/“Read & write,” friendly browsing, clean cancel, and JSON escape. Inline ${} template tokens are phase 2.
  • Drag-to-bind: Data-panel leaf to prop field makes one setProps commit. Drop-on-canvas follows catalog default-bind-target metadata later.
  • Events: labeled parameters; inferred-type literal inputs; null through JSON escape; unified SelectMenu; wired summaries, expand-to-edit, ghost add row, first-add picker cards, and wired-count badge. It lives in the Props stacked section until multi-action warrants Interactions.
  • Multi-action: sequences and per-action when conditions form a separate shared-contract lane: type widening, DSL sugar, runtime order semantics, agent guide, and eventual Interactions tab. A flow canvas is rejected for agent expressibility.

Work items

  1. S — Bug tier: padding/gap stepper and parameter labels.
  2. M — TokenControl: component family, catalog regrouping, and Retool-style section layout using N20's shared Section primitive.
  3. M — styleObject widgets: sanitizer-grammar typed controls.
  4. S — Hover preview: local override context in the canvas renderer.
  5. M — BindingPill: popover editor and affordance floor.
  6. M — Drag-to-bind: Data panel to prop fields with valid-target highlighting.
  7. M — Events rows++: complete event-editor package.
  8. L, separate lane — Multi-action contract: shared type widening, DSL, runtime, agent guide, and Interactions tab.

Sequence: 1 → 2/3 → 5 → 7 → 4/6 → 8 (contract lane).
