AI App Builder — N17 Delete Flows Plan

Locked scope and implementation sequence for the next product wave.

Decisions locked 2026-07-21.

Decisions

  • Apps: soft-delete with retention window; pages: delete from draft with server snapshot; blocks: existing undoable path.
  • Live release untouched by draft page deletion — publish remains the commit point.
  • Risk-tiered confirmations: none for blocks; named dialog for pages; type-name for apps.
  • Full-aggregate purge after retention window (pages/blocks/releases/messages/uploads/presence/signals/sessions), bounded cleanup jobs.
  • No user-facing undo after confirmation (deviates from trash/restore rec). Note: app soft-delete window still exists server-side as a safety net, but no restore UI ships in v1.
  • Agent may propose deletion; only human UI executes.

Work items

  1. M — Backend: deleteApp (soft, deletedAt), deletePage (draft removal + snapshot row), purge cron for expired apps, ownership + release guards.
  2. S — Studio UI: app card menu + type-name dialog; page list delete + named dialog; block delete unchanged.
  3. S — Cascade correctness tests (attachments/storage included); reserved-slug freeing on purge.
  4. S — Agent: propose-delete affordance in chat (card w/ deep-link to confirm UI), no delete tools.
