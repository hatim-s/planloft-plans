AI App Builder — D2 Events Platform Plan

Contract expansion for composable event sequences, guarded mutation flows, and page/application lifecycle handlers.

Decisions locked July 24, 2026 after two questionnaire rounds.

Framing

The vendored json-render fork already executes multi-action arrays serially with awaited, live parameter re-resolution, success/error branches, confirmations, and an action-observer seam. This lane chiefly widens the shared-to-DSL-to-validation-to-Studio-to-agent contract.

Decisions

  • Multi-handler shape: widen shared on to ActionBinding | ActionBinding[]. Sequences execute serially and await each action against a live snapshot.
  • Conditions and branches: add when?: VisibilityCondition, using the visible-editor grammar; evaluate each action mid-sequence. Expose existing per-action onSuccess/onError in the UI and grammar.
  • Errors: continue after an error. The emit loop catches each failure, notifies the observer and a toast, then runs remaining actions.
  • Mutation guard-assist: adding an action after runMutation inserts an editable/removable when { status == success } guard by default.
  • Payloads: add emit(event,payload) and {"$event":"field"} parameter tokens for row/list/chart identity.
  • Page/app lifecycle: add pages.handlers and apps.handlers with OCC mutations/tools and release snapshots. Page events are load/paramChange; app supports load. Declarative initialState also ships.
  • Ordering: seed initial state, compute derived values, fire once-per-visit page/app load handlers with a StrictMode guard, then launch debounced auto-queries.
  • Loop safety and DSL: thread a chain-depth cap of about 16 through execution; hard-stop and surface dev/devtools errors. Parse array-valued on:press='[{...},{...}]' while retaining single-action sugar.
  • Studio: ship the full Interactions tab now, including sequence rows, conditions, branches, payload affordance, handler editors, and event-log panel backed by the observer seam.

Cross-lane contract

D1's runMutation status mirror must settle before its handler resolves. That lets a following D2 action observe successful completion through its automatic guard.

Out of scope

Timers/intervals, a custom or cross-page event bus, page unload, and exposing raw watch fields remain deferred.

Work items

  2. M — Page/app handlers: fields, mutations/tools, OCC, release/public decode, initial state, and lifecycle ordering.
  3. S — Loop protection: chain-depth counter through execution.
  4. M — Interactions tab: sequence editor, branches, mutation guard-assist, payload affordance, and handler editors.
  5. S — Event log: surface observer dispatch, params, duration, outcome, and depth.
  6. S — Agent/docs: update the agent guide and DSL reference.

Sequence: 1 → 2/3 in parallel → 4 → 5/6.
