AI App Builder — N12 Error & Empty States Plan

Locked scope and implementation sequence for the next product wave.

Decisions locked 2026-07-21.

Decisions

  • Error-page visuals split by surface: studio theme on /apps; app theme once a release loads; neutral public shell otherwise.
  • Layered App Router boundaries: global emergency boundary + per-segment error/not-found/loading for /apps and /[appSlug].
  • Feedback: Sonner toasts for transient ops; inline errors for forms/query blocks; full-page states for fatal route failures.
  • Retry: subscriptions auto-reconnect; writes never auto-retry; external GETs bounded backoff + visible manual Retry.
  • 403 for non-owned resources, 404 for missing ids (explicit split; deviates from non-disclosure rec — backend must distinguish).

Work items

  1. S — Adopt Sonner; wire create/publish/delete/save mutations.
  2. M — Boundary files: global-error, /apps segment (error/not-found/loading skeletons), /[appSlug] segment (neutral shell + app-themed once release known).
  3. M — Backend: differentiate NotFound vs Forbidden errors in Convex functions (typed error codes); studio maps to 403/404 pages.
  4. S — Query-block retry affordances + service-down banners (agent unreachable, Convex reconnecting).
  5. S — Empty states sweep: dashboard, data panel, layers, chat, published pages without content.
