AI App Builder — Styling System Plan

Decisions locked July 18, 2026 with Hatim. A bounded styling system for generated public apps.

Code-verified facts

Block styling is hardcoded through Tailwind and shadcn tokens. Curated props already flow from definitions through the inspector, DSL, grammar, and undo. Tailwind v4 cannot discover runtime free-form strings without a safelist, and every style surface is untrusted input on public pages.

Locked decisions

D1 — Minimal, shadcn-compatible tokens. Color, radius, font family/scale, and spacing unit first; presets provide variety later.
D2 — Curated utilities only. Per-block @source inline(...) safelists permit bounded utility choices, never arbitrary class strings.
D3 — Safe advanced escape hatch. A sanitized inline style object uses a property allowlist and value regexes; raw CSS text is out.
D4 — Catalog fragment, not element field. Style props are additive metadata in definitions.ts.
D5 — Dual authored themes. Generated apps receive system-following light and dark token sets with per-app override.
D6 — Same sanitizer for agents. Agents may author the hatch but are guided toward tokens.

Work lane

Ship a usable visual language first, then add carefully bounded expressiveness.

  1. Theme document and scoped token wrapper M
    App theme, validation, setAppTheme, release snapshots, and CSS-variable roots for canvas, preview, and published apps.
  2. Theme presets and inspector panel S
    Five to eight curated presets deliver the biggest anti-dull improvement.
  3. Curated style-prop fragment S/M
    tone, size, weight, Stack surface props, and a grouped Styles inspector section.
  4. Utility-class safelist M
    Generate the safelist with @source inline, validate against it, and support it in DSL and grammar.
  5. Sanitized style-object hatch M
    Shared sanitizer, root-element spread across ten components, and Advanced UI.
  6. Agent authoring S
    Styling DSL docs, guide, and set_app_theme tool.

Sequence

1 → 2 → 3 ships useful visual value early; then 4 → 5 → 6 adds controlled power. Coordinate item 1 with N1 public serving and the light-mode canvas wrapper.
