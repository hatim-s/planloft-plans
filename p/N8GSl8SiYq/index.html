AI App Builder — Binding UX Plan

Decisions locked July 18, 2026 with Hatim. Companion to the roadmap’s resolved explorations.

Diagnosis

The underlying binding system is capable; its human surface is still raw-pointer-first.

Raw JSON pointers are the interface everywhere. The /queries/<id>/data envelope leaks; $state versus $bindState is user-facing; boolean, number, and enum props have no bind affordance; and derived values, conditions, and repeat dialects fall off happy paths into raw JSON.

Locked decisions

D1 — Friendly bound names. Show the name of the bound thing; reveal the full pointer on hover. Keep raw-path entry as a power-user toggle.
D2 — Preserve the expression model. Keep json-render expression objects and ${} template sugar. Only introduce {{ }} or @ syntax if that model becomes a proven bottleneck.
D3 — Catalog-owned direction. Per-prop bindDirection/bindIntent metadata determines the default; an advanced chip exposes overrides.
D4 — Alias at picker level. “Users” resolves to /queries/users/data; loading and error remain affordances, not a state restructure.
D5 — One stack, quick wins first. Ship the complete rethink as a single stacked-PR lane.

Work lane

One stacked PR sequence, ordered to close everyday friction before advanced modeling.

  1. Bindable data-panel JsonTree S
    Click a result field to copy it or bind it to the selected prop.
  2. Catalog binding metadata S
    Add per-prop direction and intent, remove studio defaultBindingForProp, and teach grammar the intents.
  3. Scalar bind affordances M
    Link icon and picker for boolean, number, and enum props.
  4. Browser-first PathPicker M
    Grouped friendly labels, full-path tooltips, query-envelope aliases, and raw-path mode.
  5. Structured Derived-values editor M
    Target/source/expression rows replace the JSON textarea with op-log integration.
  6. Per-block Data tab L
    Plain-language sentence rows, then evaluate dragging fields to the canvas.

Risks to preserve

  • Label collisions after a rename: the tooltip remains the precise escape hatch.
  • Repeat-scope labels need to communicate context without hiding path meaning.
  • The DSL must retain byte-stable round trips.
