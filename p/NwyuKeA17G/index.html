AI App Builder — Agent Chat UX Plan

Decisions locked July 18, 2026 with Hatim. Turn the agent transcript into a useful, reversible work surface.

SDK-verified facts

pi emits reasoning summaries via thinking_delta, full tool execution events, and accepts image input; session.ts currently retains only text_delta. runId already joins messages, opSignals, and op-log runs. A small tool gap forces whole-subtree rewrites for a single binding.

Locked decisions

D1 — Collapsible thinking summaries. Closed by default, one collapsible per item where the component kit supports it.
D2 — Tool cards in the transcript. Each card links to the canvas highlight and exposes run-level Revert.
D3 — Parts-based transport. Store a parts array on messages; retain concatenated content for compatibility.
D4 — Swappable image storage. Start with Convex files if cost permits, otherwise R2; defer the final provider choice to N5 behind a small interface.
D5 — Tool order. Prioritize set_bindings, replace_props, then query/error reads; defer screenshot inspection until styling lands.
D6 — Capability gates. Summaries-only reasoning and an image-capable model floor require a one-day runtime spike first.
D7 — Familiar UI primitives. Use Vercel AI Elements and applicable shadcn blocks, adapted to Convex-reactive data rather than useChat.

Work lane

Prove transport capabilities first; then build an inspectable and reversible conversation.

  1. Runtime spike S
    Verify thinking_delta and image input against the Codex backend.
  2. Message parts schema and emission M
    Generalize DeltaBatcher to append each part; this is the integrator change.
  3. Markdown and run grouping S
    Use AI Elements primitives in the chat transcript.
  4. Tool timeline cards M
    Canvas highlight links and runId-to-op-log Revert linkage.
  5. Thinking trace UI S
    Use the AI Elements reasoning component with D1’s collapsed default.
  6. Image attachments M
    Storage interface, upload and message attachments, service fetch/base64, and image-aware prompt/steer.
  7. Focused tool set S each
    set_bindings, replace_props, and query/error read; plan-preview later.
