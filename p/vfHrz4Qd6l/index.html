AI App Builder — D1 Data & Mutations Plan

Data-writing, real-API connectivity, and per-app secret handling with public-serving safety as the first gate.

Decisions locked July 24, 2026 after two questionnaire rounds.

Framing

Slice-2 mutation design mostly holds, but this lane replaces stale body, optimistic-timing, and header decisions while adding the secrets vault. App-data writes are external side effects: they are not undoable and remain outside the spec op-log domain.

Decisions

  • Leak fix first: strip headers from public release reads and the RSC payload now; replace client query-key headers with a config hash/revision. The proxy already injects server-side.
  • Secrets vault: per-app named secrets keyed by (appId,name), encrypted with AES-256-GCM using AAD appsecret:v1:{appId}:{name}. Values are write-only; display only last four and timestamps. Rotation overwrites.
  • References and auth: support {"$secret":"KEY"} in headers/params and an optional bearer/basic/API-key auth profile. Secrets carry owner-editable allowedOrigins[]; the proxy refuses injection for another origin.
  • Runtime resolution: a PROXY_INTERNAL_KEY-guarded Convex endpoint resolves named values at request time; crypto and master keys stay only in Convex. Releases snapshot names, not values, so rotations apply instantly.
  • Mutations: support POST, PUT, PATCH, and DELETE; enforce the configured method, follow zero redirects for writes, accept 204/empty success, and ship public-route per-app plus per-IP rate limits in the same PR.
  • Execution model: pessimistic v1: runMutation, a /mutations/{id} status mirror, and invalidates. Keep optimistic? contractual but unexecuted until append/update/remove with rollback ships together.
  • Bindable response and row actions: mirror parsed response data at /mutations/{id}/data; accept action params and merge {"$param":"name"} body leaves at fire time, including shipped $item/$index sugar.
  • Canvas, agent, and migration: writes are inert in edit canvas but live in Preview/published apps; the agent sees secret metadata only, while humans create/rotate in Settings. Block plaintext credential headers at publish time; old alpha apps may break.

Cross-lane contract

D1 must settle the mutation status mirror before the runMutation handler promise resolves. D2 sequences then reliably observe when { status == success }, including its mutation guard-assist.

Out of scope

Webhooks/inbound ingestion, OAuth flows, server-side pagination, non-JSON responses beyond empty-body success, and undo for app-data writes.

Work items

  1. S — Leak redaction: remove public header exposure and use a query-key hash/revision.
  2. M — Secrets vault: table, scoped crypto, referenced-by guard, and write-only Settings UI.
  3. M — Secret resolution: secret/auth references, allowed-origin enforcement, and guarded Convex decrypt path for draft and public requests.
  4. M — Mutations runtime: action, params/body merge, method/redirect/204/rate-limit rules, and settled status/data mirrors.
  5. S — Configuration UI: mutation editor, secret affordance, and publish-time plaintext-header block.
  6. S — Agent: secret-name-only listing tool plus mutation authoring grammar and guide.
  7. Fast-follow M — Optimistic verbs: append/update/remove with automatic rollback.

Sequence: 1 → 2/3 → 4 → 5/6 → 7. No external dependency; coordinate the handler-settle contract with D2.
