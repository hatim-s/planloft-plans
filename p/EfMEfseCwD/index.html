Research-backed lanes

Two focused tracks build on the current repeat substrate and bolt’s path-aware store primitives.

ELane E — List block with item contextS → L

The repeat substrate mostly exists: element-level repeat={statePath,key}, ${field} item templates, and repeat-ancestor-scoped $bindItem already support per-item fields inside repeated Cards. The missing piece is a first-class List block and item-scoped ergonomics.

  • E1 · S — Lock the List contract: element-level repeat, one item-template child, statusPath plus empty/loading/error states, key policy, and accessible semantics.
  • E2 · M — Build it: definition, component, registry, palette and inspector support, plus render/DSL tests. Canvas selection and DnD over repeated DOM are the main risk.
  • E3 · M — Docs/grammar: add the List example and conventions to DSL.md, catalog grammar, and prompt examples; reconcile the existing Table/Chart documentation drift.
  • E4 · M — Scoped-data tests: nested item paths, $bindItem, per-item Table/Chart, state transitions, and key reorder.
  • E5 · L — Item-identity action payloads: keep on:select identity work separate; json-render currently turns $item params into paths and emit has no payload channel.
  • E6 · L, deferred — Virtualization: variable-height charts, focus, and SSR/hydration complexity.

First increment: E1 → E2 layered on existing repeat, with E4 tests. Charts and tables per item work via $bindItem from day one.

FLane F — Render perf: use bolt properlyS → M

Reproduced root cause: bolt’s path-granular notifications are discarded twice. The preview adapter subscribes at the store root and rebuilds a whole-state snapshot per write; json-render then sends it through one changing React context, defeating ElementRenderer memoization. Bolt already has the needed path subscriptions, topologically settled derived paths, and reference-stable partial writes.

  • F1 · S — Instrument: add a preview render-count probe to confirm the issue and measure the win.
  • F2 · S — Expose granularity: add RuntimeStores.subscribePath(pointer) and getPath(pointer) over the two-tier stores.
  • F3 · M — Granular binding: create bolt-backed useRuntimeValue(pointer) with useSyncExternalStore and memoized block shells that subscribe only to bound pointers. Blocks must stop reading json-render’s StateContext for values.
  • F4 · M — Derived bindings: register bolt derive(target, [queryResultPointer], compute) at spec assembly, with disposal lifecycle, so refreshes recompute only dependents.
  • F5 · L, optional — Upstream json-render: pursue a per-path subscription API only if stock components should inherit the fix.

F1 → F4 solves the reported typing case in-repo and showcases bolt’s derived-dependency flow ahead of N6.
