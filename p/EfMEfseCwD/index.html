AI App Builder — Progress & Roadmap

Builder core shipped. Status verified against code on main.

A human and an AI agent co-edit JSON-defined applications live: specs render to React at runtime, and both collaborators use the same granular mutation model.

Shipped

The product foundation and the first builder-quality wave are in the codebase.

Foundation

  • Monorepo scaffold with frozen shared contracts
  • Studio builder: app/page creation, palette, inspector, chat, publish snapshot
  • Agent service translating Pi tools to DSL and live canvas blocks
  • Preview route and nine-block catalog

Data layer

  • QueryRuntime wired end-to-end with reactive {$state} params
  • Auto/manual modes, app-scoped queries, SSRF-guarded CORS proxy
  • Data panel with live results and runQuery/setState actions
  • Table and Chart blocks with validation and style-injection guards

Editor

  • Drag-and-drop canvas and Layers hierarchy through undo tracking
  • Structured event, visible, and repeat editors
  • PathPicker, TemplateInput, Combobox, and repeat-scoped autocomplete
  • Scalar-safe literals and accessibility pass

Undo

  • Session-scoped op log with capture, inverses, preconditions, and cap
  • Cmd+Z and header Undo button
  • Granular and per-run agent reverts
  • Publish clears the log; redo remains intentionally out of v1

Agent

  • SDK-native steering: deliver/abort, run registry, idle eviction
  • Human-edit injection with per-page conflict tracking
  • Multi-page awareness and mutation attribution
  • Op-signal feed, block versions, and eval harness

List + rendering

  • List contract, itemScope, and per-item context
  • Query-backed source with loading, empty, error, and retry states
  • $item/$index action sugar with reserved-segment hardening
  • Path-granular preview rendering and derived page state
