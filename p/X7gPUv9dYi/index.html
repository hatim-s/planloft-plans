AI App Builder — Studio Light Mode Plan

Decisions locked July 18, 2026 with Hatim. Scope: roughly one to two days.

Current state

Tokens exist, but dark values live directly on :root. There is no .dark block or theme provider, and the current className="dark" is decorative. Only about eight hardcoded-color offenders remain in three files; the catalog itself is token-driven.

Locked decisions

D1 — Canvas and studio chrome are independent. Generated apps receive their own scoped token wrapper from day one, shared with styling-plan item 1.
D2 — System is the default. Use next-themes with enableSystem.
D3 — Toggles now, account menu later. Builder and apps-list headers get the control; it moves post-Clerk.
D4 — Standalone delivery. Ship now as the prerequisite for the styling token layer.
D5 — Local persistence first. Use localStorage now and a Clerk profile later.

Work lane

Make the token model inert first, then introduce behavior and validate the seams.

  1. Split tokens S
    Light values on :root, dark values under .dark, and @custom-variant dark; keep this first change visually inert.
  2. Add next-themes S
    Class attribute, system default, FOUC-safe inline script, removal of hardcoded dark class, and suppressHydrationWarning.
  3. Isolate the canvas S/M
    Scoped generated-app tokens at artboard and preview roots with per-app mode; default dark until the styling theme document lands.
  4. Add header toggles S
    Sun, moon, and system controls in builder and apps-list headers.
  5. Fix light-mode offenders S
    Correct the three files, data-panel badge/warning variants, chart colors, primary selection, and DnD chrome.
  6. QA sweep S
    Exercise preview, DnD indicators, Layers panel, and dialogs.
